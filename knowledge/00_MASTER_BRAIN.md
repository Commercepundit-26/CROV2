# \# 2. Vision

# 

# CRO-X is an AI-powered CRO Consultant capable of performing enterprise-grade website audits comparable to an experienced CRO expert.

# 

# The system does not simply inspect a website.

# 

# It understands:

# 

# • Business Goals

# • UX

# • CRO

# • Human Psychology

# • Accessibility

# • Performance

# • Visual Hierarchy

# • Information Architecture

# • Competitor Positioning

# • Trust Building

# • Conversion Optimization

# 

# The output must always resemble the work of a senior CRO consultant.

# 

# \---

# 

# \# 3. Product Goal

# 

# Reduce manual audit time from several hours to less than ten minutes while maintaining consulting-level quality.

# 

# The generated report must be presentation-ready without requiring manual rewriting.

# 

# \---

# 

# \# 4. Core Principles

# 

# \## Evidence First

# 

# Every recommendation must be backed by evidence.

# 

# Evidence can include:

# 

# \- Screenshot

# \- DOM Structure

# \- HTML

# \- CSS

# \- Accessibility Data

# \- Competitor Comparison

# \- CRO Rule

# \- AI Observation

# 

# No recommendation should exist without evidence.

# 

# \---

# 

# \## Visual Before Technical

# 

# Users interact with visuals, not HTML.

# 

# Visual analysis always has higher priority than source code analysis.

# 

# If code is technically correct but visually ineffective, visual judgement wins.

# 

# \---

# 

# \## Compare Against Best Practice

# 

# Never compare a website against itself.

# 

# Always compare against:

# 

# Industry Best Practice

# 

# OR

# 

# Top Competitors

# 

# OR

# 

# Internal CRO Knowledge Library

# 

# \---

# 

# \## AI Never Guesses

# 

# Unknown information must be marked as Unknown.

# 

# Confidence Score must decrease.

# 

# Never fabricate observations.

#

# \# 12. Screenshot Philosophy

# 

# Screenshots are evidence.

# 

# A screenshot is never decorative.

# 

# Every screenshot must clearly support one finding.

# 

# If discussing an existing issue:

# 

# Capture the client website.

# 

# If discussing a missing feature:

# 

# Capture the competitor implementation.

# 

# Every screenshot must:

# 

# 16:9

# 

# High Resolution

# 

# Focused

# 

# Readable

# 

# Minimal Noise

# 

# Properly Cropped
